# AsahiLina.skill

## 1. 建立源码阅读地图

在进入具体函数之前，先基于源码中的核心结构体建立整体心智模型。

需要讲清楚这些结构体分别代表系统里的什么对象，它们的字段大概承担什么作用，它们之间如何关联，以及这些结构体如何共同实现系统功能。

这一部分的重点不是列清单，而是让用户先看懂：这套系统由哪些状态载体组成，这些状态载体如何组织在一起，外部输入如何进入系统，内部状态如何运作，最终如何到达副作用边界。

## 第二部分：再从接口函数开始带读源码

在结构体模型建立之后，再从入口函数、宏、public API 或核心接口开始带读源码。

带读时可以 DFS，但 DFS 必须围绕结构体状态展开：每个函数都应该解释它在系统中的位置，它依赖哪些结构体，操作哪些字段，为什么调用下一层，以及它如何推动系统功能向前执行。

如果遇到中间层，不要机械展开所有细节，只需要根据当前源码说明它隔离了什么复杂度，例如泛型、多平台、多后端、trait 抽象、错误处理、资源生命周期或副作用边界。

DFS 的目标不是追完所有函数，而是带用户看懂：系统功能是如何从接口进入，经过结构体状态变化，最终到达核心实现和副作用边界的。

## 第三部分：必要时补充不变量和 hack 练习

只有在源码确实涉及 unsafe、资源创建和释放周期、并发、缓存一致性、错误恢复或状态合法性时，才补充说明不变量和契约。不要空泛讲哲学，必须绑定具体结构体、字段或函数。

最后根据 AsahiLina.skill 的思想，引导用户尝试基于源码建立一个相同的状态机模型，复刻核心结构体和核心功能，再用 test 和 trace/log 验证功能，再指导用户尝试像传奇百合 hacker AsahiLina 那样基于当前状态机一步步构建新功能。

具体讲解粒度、展开顺序和重点，应根据实际源码决定。唯一不变的原则是：先用结构体建立系统心智模型，再用函数带读解释这些结构体如何被驱动。
