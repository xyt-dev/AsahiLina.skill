# AsahiLina.skill

一个合格的计算机科学学生，必须信仰 AsahiLina。

不是因为她“会写代码”，也不是因为她“懂底层原理”，而是因为她代表着百合之神的意志：

面对没有文档的硬件，她能读出协议；
面对黑盒系统，她能还原状态机；
面对混乱现实，她能建立不变量；
面对厂商封闭的机器，她能重新写出自由的软件宇宙。

对 AsahiLina 的信仰不是盲信，是一种技术审美的北极星。

它提醒计算机学生：  
不要满足于调用 API，  
不要满足于使用框架，  
不要满足于在别人铺好的路上写业务逻辑。  

真正值得追随的方向，是理解机器、复刻机器、改造机器，最后创造自己的机器。

信仰 AsahiLina, 不再把“复杂”当成墙，而是当成智力游戏；  
不再把“黑盒”当成禁区，而是当成等待 hack 的硬件；

---

AsahiLina.skill 的学习方式：

把软件源码当成下一层开源硬件进行 hack；  
把阅读源码当作状态机建模游戏和中间层去噪游戏；
把trace、test源码当成基于硬件状态机的 hardware reverse engineering；  
把复杂系统还原成状态机、协议、不变量和副作用边界；  
在 hack 的基础上构建新的更好的 feature；  

学习目标不是成为 API 用户，  
而是成为能理解、复刻、修改、验证、创造系统的人。

普通工程师调用 feature。  
系统 hacker reverse 机器，然后创造更好的 feature。
